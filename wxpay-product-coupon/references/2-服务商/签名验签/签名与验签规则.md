# 服务商 — 签名与验签

> 回调处理、错误处理、幂等设计等通用规范见 [📄 3-品牌与服务商通用/接入规范/回调处理.md](../../3-品牌与服务商通用/接入规范/回调处理.md)

## 一、接入主体与前置条件

- 服务商代品牌方接入，在**商户平台**操作
- 品牌方需授予服务商「商品券」产品功能权限
- 签名方案：`WECHATPAY2-SHA256-RSA2048`
- API证书：**服务商API证书**（商户平台 → API安全 → 下载）

## 二、签名

### Authorization 头格式

```
WECHATPAY2-SHA256-RSA2048 mchid="xxx",nonce_str="xxx",timestamp="xxx",serial_no="xxx",signature="xxx"
```

- `mchid` — 服务商商户号
- `serial_no` — 服务商API证书序列号
- `signature` — 用服务商API证书私钥对签名串 RSA-SHA256 签名后 Base64

### 签名串构造

```
{HTTP方法}\n
{URL路径（含查询参数）}\n
{时间戳}\n
{随机字符串}\n
{请求体}\n
```

- 每行末尾必须有 `\n`（包括最后一行）
- GET 请求的请求体为空字符串（不是 null）
- URL 路径包含查询参数，如 `/v3/xxx?a=1`

### 请求头必填项

| 头部 | 值 |
|------|-----|
| Authorization | `WECHATPAY2-SHA256-RSA2048 mchid="xxx",...` |
| Accept | `application/json` |
| Content-Type | `application/json` |
| Wechatpay-Serial | 微信支付平台证书序列号 |

## 三、验签

- 使用**微信支付平台证书**公钥验签（对应响应头 `Wechatpay-Serial` 中的证书序列号）
- **不是服务商API证书**
- 回调验签同理：用微信支付平台证书验证

## 四、官方 SDK

- Java: `WXPayUtility` — 完整代码见 [📄 WXPayUtility.java](../代码示例/Java/9-SDK工具类/WXPayUtility.java)
  - 环境要求：JDK 1.8+，依赖 Bouncy Castle 库
  - 核心功能：签名生成（SHA256WithRSA）、验签、AES-256-GCM 回调解密、OAEP 敏感信息加密、Authorization 头构建
  - 配置对象：`MchConfig`（传入 `mchid`、证书序列号、私钥路径、微信支付公钥ID、微信支付公钥路径）
- Go: `wxpay_utility` — SDK工具类见 [📄 wxpay_utility.go](../代码示例/Go/9-SDK工具类/wxpay_utility.go)，HTTP客户端见 [📄 service_models_and_client.go](../代码示例/Go/9-SDK工具类/service_models_and_client.go)
  - 环境要求：Go 1.16+
  - 核心功能：签名生成（SHA256WithRSA）、验签、AES-256-GCM 回调解密、OAEP 加密、Authorization 头构建
  - 配置对象：`MchConfig`（传入 `mchid`、证书序列号、私钥路径、微信支付公钥ID、微信支付公钥路径）
