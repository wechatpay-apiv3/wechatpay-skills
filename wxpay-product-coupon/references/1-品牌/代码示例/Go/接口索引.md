## 接入模式

品牌直连模式，品牌方自行接入微信支付商品券。

## 前置依赖

- 工具库：`wxpay_brand_utility`，处理签名、验签、回调解密
- 包名：`main`
- 引用路径：`demo/wxpay_brand_utility`

## SDK工具类

[`brand_models_and_client.go`](./9-SDK工具类/brand_models_and_client.go)（位于 `9-SDK工具类/` 目录），所有业务代码都依赖此文件。

包含以下内容：

- HTTP 客户端：自动处理请求签名和响应验签
- 核心方法：封装了商品券全部 API 的 HTTP 调用（签名、发送、验签、解析），业务代码通过调用这些方法完成请求
- 配置结构体：`wxpay_brand_utility.BrandConfig`，包含 `BrandId`、`CertSerialNo`、`PrivateKey`、`WechatPayPubKeyId`、`WechatPayPubKey`
- 枚举定义：
  - `ProductCouponScope` — ALL（全场）/ SINGLE（单品）
  - `ProductCouponType` — NORMAL（满减）/ DISCOUNT（折扣）/ EXCHANGE（兑换）
  - `UsageMode` — SINGLE（单券）/ PROGRESSIVE_BUNDLE（多次优惠）
  - `CouponCodeMode` — WECHATPAY（微信支付生成）/ UPLOAD（品牌方预上传）/ API_ASSIGN（品牌方指定）

## 初始化参数（所有文件通用）

```go
config, err := wxpay_brand_utility.CreateBrandConfig(
    "xxxxxxxx",                    // brand_id（品牌ID）
    "1DDE55AD98Exxxxxxxxxx",       // 品牌API证书序列号
    "/path/to/apiclient_key.pem",  // 品牌API证书私钥文件路径
    "PUB_KEY_ID_xxxxxxxxxxxxx",    // 微信支付公钥ID
    "/path/to/wxp_pub.pem",        // 微信支付公钥文件路径
)
```

> 用户提供具体值后，直接替换上方占位符即可；其余代码逻辑不要改动。
