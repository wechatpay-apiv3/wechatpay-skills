## 接入模式

服务商模式，服务商代品牌方接入微信支付商品券。

## 前置依赖

- 工具库：`WXPayUtility`，处理签名、验签、回调解密
- 包名：`com.java.demo`
- HTTP 库：OkHttp
- JSON 库：Gson

## 与品牌直连版的差异

| 差异项 | 品牌直连 | 服务商 |
|--------|---------|--------|
| 工具库 | `WXPayBrandUtility` | `WXPayUtility` |
| 签名前缀 | `WECHATPAY-BRAND-SHA256-RSA2048` | `WECHATPAY2-SHA256-RSA2048` |
| 身份标识 | `brand_id` | `mchid`（商户号），请求体中额外传 `brand_id` |
| API 路径 | `/brand/marketing/product-coupon/...` | `/v3/marketing/partner/product-coupon/...` |
| 多次优惠批次字段 | `StockForCreate` | `StockBundleForCreate`（服务商专有模型） |

## SDK工具类

[`ServiceModelsAndClient.java`](./9-SDK工具类/ServiceModelsAndClient.java)（位于 `9-SDK工具类/` 目录），所有业务代码都依赖此文件。

包含以下内容：

- HTTP 客户端：自动处理请求签名（SHA256-RSA2048）和响应验签
- 核心方法：封装了商品券全部 API 的 HTTP 调用（签名、发送、验签、解析），业务代码通过调用这些方法完成请求
- 数据模型：34+ 内部类，含服务商专有模型 `StockBundleForCreate` / `StockBundleEntity`
- 枚举定义：
  - `ProductCouponScope` — ALL（全场）/ SINGLE（单品）
  - `ProductCouponType` — NORMAL（满减）/ DISCOUNT（折扣）/ EXCHANGE（兑换）
  - `UsageMode` — SINGLE（单券）/ PROGRESSIVE_BUNDLE（多次优惠）
  - `CouponCodeMode` — WECHATPAY（微信支付生成）/ UPLOAD（品牌方预上传）/ API_ASSIGN（品牌方指定）

## 初始化参数（所有文件通用）

```java
new XxxJava(
  "19xxxxxxxx",                  // mchid（服务商商户号）
  "1DDE55AD98Exxxxxxxxxx",       // 商户API证书序列号
  "/path/to/apiclient_key.pem",  // 商户API证书私钥文件路径
  "PUB_KEY_ID_xxxxxxxxxxxxx",    // 微信支付公钥ID
  "/path/to/wxp_pub.pem"         // 微信支付公钥文件路径
);
// 请求体中还需设置 request.brandId = "120344"; // 品牌ID
```

> 用户提供具体值后，直接替换上方占位符即可；其余代码逻辑不要改动。
