# MAB关系说明文档 — brand_id、mchid、appid

> 本文档适用于品牌直连和服务商两种接入模式。
> 理解三者关系是正确传参的前提，尤其在服务商模式下容易混淆。

## 使用指引

当用户遇到以下场景时，应参考本文档回答：
- 询问 brand_id、mchid、appid 三者的关系或区别
- 不清楚品牌直连与服务商模式下分别该传哪些参数
- 遇到 `INVALID_REQUEST: AppID非法`、`NO_AUTH` 等参数关联类报错
- 询问 appid 如何绑定/关联品牌
- 询问服务商如何获得品牌授权

## 三者关系概览

```
appid1 ──┐                          ┌── appid1
appid2 ──┤                          ├── appid2
  ...  ──┤                          ├──  ...
         │                          │
      mchid 1 ──╮      ╭── brandid 1
         │       ╲    ╱        │
         │        ╲  ╱         │
         │         ╳           │
         │        ╱  ╲         │
         │       ╱    ╲        │
      mchid 2 ──╯      ╰── brandid 2
         │                          │
appid1 ──┤                          ├── appid1
appid2 ──┤                          ├── appid2
  ...  ──┘                          └──  ...

(服务商侧appid)  (服务商)  (品牌)  (品牌侧appid)
```

| 关系 | 类型 | 说明 |
|------|------|------|
| **mchid ↔ brand_id** | 多对多 | 一个服务商可服务多个品牌，一个品牌也可授权给多个服务商 |
| **mchid ↔ appid** | 一对多 | 一个服务商下挂多个 appid（服务商自己的小程序、公众号等） |
| **brand_id ↔ appid** | 一对多 | 一个品牌下挂多个 appid（品牌自己的小程序、公众号等） |

> ⚠️ 服务商侧的 appid 和品牌侧的 appid 是**两套独立的 appid**，分属不同主体。

## 品牌直连创券前必须完成的前置条件

品牌直连模式下，只涉及 **brand_id** 和 **品牌 appid**，不涉及 mchid。

### ① AppID 关联品牌（brand_id ↔ appid）

创券时 `stock.notify_config.notify_appid` 字段必须传一个已关联品牌的 AppID。

**如何关联**（二选一）：

| 绑定路径 | 操作位置 |
|---------|---------|
| 路径一 | 品牌经营平台 → 账号管理 → 下单 appid 管理 |
| 路径二 | 品牌经营平台 → 商家名片 → 交易连接名片 |

**不关联会怎样**：创券接口返回 **`INVALID_REQUEST: AppID非法`**（错误码 `268578704`），无法创建商品券。

### ② 签名参数配置

| 参数 | 在请求中的位置 | 说明 |
|------|-------------|------|
| brand_id | Authorization 签名头 `brand_id="xxx"` | 品牌身份标识，**不需要**在请求体中传 |
| 品牌 appid | 请求体 `notify_config.notify_appid` | 需已完成上述关联 |

签名方案：`WECHATPAY-BRAND-SHA256-RSA2048`

## 服务商创券前必须完成的前置条件

服务商模式下，brand_id、mchid、appid 三者都涉及，**且有两项前置条件必须同时满足，缺一不可**。

### ① 品牌授权服务商（brand_id ↔ mchid）

品牌方必须在品牌经营平台授予服务商「商品券」产品功能权限。

**如何授权**：品牌方登录品牌经营平台，在合作管理中对服务商授权「商品券」产品功能。授权流程见 [服务商申请品牌授权](https://pay.weixin.qq.com/doc/v3/partner/4016026183)。

**不授权会怎样**：服务商调用任何商品券 API 均返回 **`NO_AUTH: 服务商没有此接口权限`**（错误码 `268579803`），所有接口不可用。

### ② AppID 关联品牌（brand_id ↔ appid）

创券时 `stock.notify_config.notify_appid` 字段必须传一个已关联品牌的 AppID（可以是服务商 appid 或品牌 appid，但必须由品牌方在品牌经营平台完成授权关联）。

**如何关联**（二选一，均需品牌方在品牌经营平台操作）：

| 绑定路径 | 操作位置 |
|---------|---------|
| 路径一 | 品牌经营平台 → 账号管理 → 下单 appid 管理 |
| 路径二 | 品牌经营平台 → 商家名片 → 交易连接名片 |

> 💡 服务商可通过 API `POST /v3/brand/partner/cardtemplate/traderelation` 创建交易连接名片，但 AppID 关联品牌仍需品牌方在品牌经营平台确认。

**不关联会怎样**：创券接口返回 **`INVALID_REQUEST: AppID非法`**（错误码 `268578704`），无法创建商品券。

### ③ 签名参数配置

| 参数 | 在请求中的位置 | 说明 |
|------|-------------|------|
| mchid | Authorization 签名头 `mchid="xxx"` | 服务商身份标识 |
| brand_id | **请求体**中传入 | 指定为哪个品牌操作，**服务商必填** |
| appid | 请求体 `notify_config.notify_appid` | 服务商 appid 或品牌 appid，需已完成上述关联 |

签名方案：`WECHATPAY2-SHA256-RSA2048`
