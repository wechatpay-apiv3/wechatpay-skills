# 品牌直连 — 签名与验签

> 回调处理、错误处理、幂等设计等通用规范见 [📄 3-品牌与服务商通用/接入规范/回调处理.md](../../3-品牌与服务商通用/接入规范/回调处理.md)

## 一、接入主体与前置条件

- 品牌方自行接入，在**品牌经营平台**注册和操作
- 签名方案：`WECHATPAY-BRAND-SHA256-RSA2048`
- API证书：**品牌API证书**（品牌经营平台 → API安全 → 下载）

## 二、签名

### Authorization 头格式

```
WECHATPAY-BRAND-SHA256-RSA2048 brand_id="xxx",nonce_str="xxx",timestamp="xxx",serial_no="xxx",signature="xxx"
```

- `brand_id` — 品牌ID（品牌经营平台注册后生成）
- `serial_no` — 品牌API证书序列号
- `signature` — 用品牌API证书私钥对签名串 RSA-SHA256 签名后 Base64

### 签名串构造

```
{HTTP方法}\n
{URL路径（含查询参数）}\n
{时间戳}\n
{随机字符串}\n
{请求体}\n
```

- 每行末尾必须有 `\n`（包括最后一行）
- GET 请求的请求体为空字符串（不是 null）
- URL 路径包含查询参数，如 `/brand/xxx?a=1`

### 请求头必填项

| 头部 | 值 |
|------|-----|
| Authorization | `WECHATPAY-BRAND-SHA256-RSA2048 brand_id="xxx",...` |
| Accept | `application/json` |
| Content-Type | `application/json` |
| Wechatpay-Serial | 微信支付公钥ID |

## 三、验签

- 使用**微信支付公钥**验签（对应响应头 `Wechatpay-Serial` 中的公钥ID）
- **不是品牌API证书**
- 回调验签同理：用微信支付公钥验证

## 四、官方 SDK

- Java: `WXPayBrandUtility` — 完整代码见 [📄 WXPayBrandUtility.java](../代码示例/Java/9-SDK工具类/WXPayBrandUtility.java)
  - 环境要求：JDK 1.8+，依赖 Bouncy Castle 库
  - 核心功能：签名生成（SHA256WithRSA）、验签、AES-256-GCM 回调解密、OAEP 敏感信息加密、Authorization 头构建
  - 配置对象：`BrandConfig`（传入 `brandId`、证书序列号、私钥路径、微信支付公钥ID、微信支付公钥路径）
- Go: `wxpay_brand_utility` — SDK工具类见 [📄 wxpay_brand_utility.go](../代码示例/Go/9-SDK工具类/wxpay_brand_utility.go)，HTTP客户端见 [📄 brand_models_and_client.go](../代码示例/Go/9-SDK工具类/brand_models_and_client.go)
  - 环境要求：Go 1.16+
  - 核心功能：签名生成（SHA256WithRSA）、验签、AES-256-GCM 回调解密、OAEP 加密、Authorization 头构建
  - 配置对象：`BrandConfig`（传入 `brandId`、证书序列号、私钥路径、微信支付公钥ID、微信支付公钥路径）
